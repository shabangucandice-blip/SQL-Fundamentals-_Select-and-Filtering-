# SQL-Fundamentals-_Select-and-Filtering-
Hand- written exercise


This BrightLearn exercise is a hands-on, traditional "pen-and-paper" drill designed to master the fundamentals of SQL data retrieval and filtering.
Instead of relying on a computer to catch errors, We are required to manually write queries using SELECT, WHERE, and ORDER BY, and then manually "draw" the resulting data tables.

Builds "Mental Execution": We are forced to act as the database engine. By manually tracing which rows meet a WHERE condition, We develop a deep intuition for how SQL actually logic-checks data.
Syntax Muscle Memory: Without "Auto-complete" or "IntelliSense" to help us. We have to memorise the exact order of clauses (SELECT → FROM → WHERE → ORDER BY). This makes us a much faster and more accurate coder when We return to a keyboard.

Attention to Detail: When We have to physically draw a table, We  become hyper-aware of "null" values, data types, and specific column names. It eliminates the "trial and error" habit, encouraging us to get the code right the first time.

Interview Readiness: Many technical interviews for data roles involve "whiteboarding"—writing code on a board without a computer. Practicing on paper is the best way to prepare for that pressure.
